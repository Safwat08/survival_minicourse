# DeepHIT Loss

DeepHIT (Lee et al., 2018) is a deep learning survival model that handles both single and competing risks without assuming any specific hazard form.

## Model Output

Instead of a continuous hazard, DeepHIT outputs a discrete joint distribution over event type *k* and time bin *t*:

```
p(T = t, K = k | x)
```

The time axis is discretized into bins.

## Loss Components

The total loss is a weighted sum of two terms:

### 1. Log-likelihood Loss

```
L1 = -Σ_i [ δ_i * log p(T_i, K_i | x_i) + (1 - δ_i) * log S(T_i | x_i) ]
```

### 2. Ranking Loss (Cause-specific C-index)

Encourages correct ordering of predicted risks (the C-index is the standard ranking/discrimination metric — see [`5.1`](../05_metrics/5.1_c_index.md)):

```
L2 = -Σ_{i,j: T_i < T_j, δ_i=1} exp( -(F_i - F_j) / σ )
```

where F_i is the predicted cumulative incidence for subject i.

### Combined

```
L = L1 + α * L2
```

## Advantages

- No proportional hazards assumption
- Handles competing risks natively
- Flexible time discretization
- Directly optimizes both likelihood and ranking

## Implementation

**pycox** is the reference implementation — `DeepHitSingle` for one event, `DeepHit` for competing risks (see [`references/6.1_pycox.md`](../references/6.1_pycox.md)). Like all discrete-time models it needs a label transform to bin the time axis:

```python
from pycox.models import DeepHitSingle

labtrans = DeepHitSingle.label_transform(num_durations=20)
y_train = labtrans.fit_transform(durations, events)

model = DeepHitSingle(net, optimizer,
                      alpha=0.2, sigma=0.1,        # alpha weights L1 vs L2; sigma is the ranking scale
                      duration_index=labtrans.cuts)
model.fit(x_train, y_train, epochs=100)
surv = model.predict_surv_df(x_test)               # full S(t) per subject
```

For competing risks, swap in `pycox.models.DeepHit` and provide event-type labels k ∈ {1, …, K}.

**auton-survival** offers an alternative deep competing-risks model, Deep Survival Machines (DSM), which mixes parametric distributions instead of a discrete grid (see [`references/6.5_auton_survival.md`](../references/6.5_auton_survival.md)):

```python
from auton_survival.models.dsm import DeepSurvivalMachines
model = DeepSurvivalMachines(layers=[100]).fit(features, outcomes.time, outcomes.event)
```

> **The two hyperparameters that matter.** `alpha` trades likelihood (calibration) against ranking (discrimination/C-index); `sigma` controls how sharply the ranking term penalizes mis-ordered pairs. Tune both — the defaults rarely win. This loss is the natural choice when [`01_background/1.4_multi_events_and_competing_risks.md`](../01_background/1.4_multi_events_and_competing_risks.md) applies.
