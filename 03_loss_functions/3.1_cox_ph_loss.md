# Cox Proportional Hazards Loss

The Cox PH loss is the most widely used loss function in survival analysis. It is based on the partial likelihood introduced by Cox (1972).

## Model

The hazard for subject *i* at time *t*:

```
h(t | x_i) = h_0(t) * exp(β^T x_i)
```

The baseline hazard h_0(t) is left unspecified — the model is *semi-parametric*.

## Partial Likelihood

### The idea

A *full* likelihood for survival data would need the baseline hazard h_0(t) — the actual shape of risk over time. Cox's trick is to estimate the covariate effects β **without ever specifying h_0(t)**, by using only the part of the information that does not depend on it. That reduced quantity is the *partial* likelihood.

It works by asking, each time an event occurs: *given that exactly one subject in the risk set failed at this moment, what is the probability it was this particular subject?*

```
P(subject i fails | one failure at t_i in R(t_i)) = exp(β^T x_i) / Σ_{j ∈ R(t_i)} exp(β^T x_j)
```

Each subject's hazard is h_0(t)·exp(β^T x_j), so the shared h_0(t_i) **cancels** between numerator and denominator — that cancellation is exactly what removes the baseline hazard. The ratio is a *softmax over the risk set*, which is why the deep-learning version is just a log-softmax loss.

### The loss

Multiply these per-event conditional probabilities across all observed events and take the negative log:

```
L(β) = -Σ_{i: δ_i=1} [ β^T x_i - log( Σ_{j ∈ R(t_i)} exp(β^T x_j) ) ]
```

where R(t_i) is the *risk set* — subjects still at risk just before time t_i. A censored subject never gets its own numerator term, but it still appears in the risk sets (denominators) of earlier events, so its follow-up still contributes.

> **Example.** Three patients: A fails first, then B, while C is censored. The partial likelihood is [A's softmax over {A, B, C}] × [B's softmax over {B, C}]. C contributes only to denominators, never a numerator.

### Why "partial"

It is *partial* because it deliberately discards information: it uses only the **order** in which events occur (who failed before whom), not the actual event times — those live in the baseline hazard that was dropped. That is also why a Cox model outputs a **ranking** (a risk score) rather than absolute probabilities, and why its natural evaluation metric is the rank-based [C-index](../05_metrics/5.1_c_index.md).

## Properties

- Does not require specifying the baseline hazard
- Proportional hazards assumption: the hazard ratio between subjects is constant over time
- Outputs a log-hazard (risk score), not a probability

## In Deep Learning

Replace β^T x with a neural network output φ(x). The loss remains the same (negative log partial likelihood). Used in **DeepSurv**.

## Implementation

The Cox loss is available at every altitude. Pick by how much you own the training loop.

**torchsurv** — raw differentiable loss for a custom PyTorch model (the SSL3D case; see [`references/6.3_torchsurv.md`](../references/6.3_torchsurv.md)):

```python
from torchsurv.loss import cox

log_hz = model(x)                 # network output: one log-hazard per subject
loss = cox.neg_partial_log_likelihood(log_hz, events, times)
```

**pycox** — packaged DeepSurv (`CoxPH`) / memory-efficient `CoxCC` (see [`references/6.1_pycox.md`](../references/6.1_pycox.md)):

```python
import pycox.models as models, torchtuples as tt
net = tt.practical.MLPVanilla(in_features, [64, 64], out_features=1)
model = models.CoxPH(net, tt.optim.Adam)
model.fit(x_train, (durations, events), epochs=100)
```

**lifelines** — classical Cox on tabular data, full statistical summary (see [`references/6.2_lifelines.md`](../references/6.2_lifelines.md)):

```python
from lifelines import CoxPHFitter
CoxPHFitter().fit(df, duration_col="T", event_col="E").print_summary()
```

**scikit-survival** — regularized/penalized Cox inside an sklearn pipeline (see [`references/6.4_scikit_survival.md`](../references/6.4_scikit_survival.md)):

```python
from sksurv.linear_model import CoxPHSurvivalAnalysis
CoxPHSurvivalAnalysis().fit(X_train, y_train)   # y_train is a structured (event, time) array
```

> **Note on risk-set scope.** The exact partial likelihood sums over the *global* risk set. With mini-batches, the risk set is restricted to the batch — large batches reduce this approximation error. `CoxCC` sidesteps it via case-control sampling.
