# torchsurv

`torchsurv` is a lightweight PyTorch library providing survival loss functions and metrics as native PyTorch operations, designed to integrate easily into custom training loops.

## Installation

```bash
pip install torchsurv
```

## Key Features

- **Losses**: Cox, Weibull (AFT), flexible hazard NLL, and momentum/contrastive tools
- **Metrics**: C-index, Brier score, and AUC implemented with native PyTorch tensors and GPU support
- Integrates directly with any PyTorch model

## Common Usage

```python
import torch
from torchsurv.loss import cox
from torchsurv.metrics.cindex import ConcordanceIndex

# Training
log_hz = model(x)  # your network output
loss = cox.neg_partial_log_likelihood(log_hz, events, times)

# Evaluation
cindex = ConcordanceIndex()
c = cindex(log_hz, events, times)
```

Metrics should normally be computed on accumulated validation/test predictions, not
averaged from separate mini-batch metric values: comparable pairs and censoring
weights can cross batch boundaries. The Cox training loss has the same risk-set
caveat; see [`2.1`](../02_loss_functions/2.1_cox_ph_loss.md#key-takeaways-for-applied-ml-scientists).

## Best For

- Custom PyTorch survival architectures
- 3D imaging models (CNNs, ViTs) where you own the forward pass
- When you need GPU-compatible survival metrics without leaving PyTorch
- Integrating survival loss into SSL or multi-task pipelines
