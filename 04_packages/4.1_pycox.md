# pycox

`pycox` is a Python package for deep learning survival models built on top of PyTorch and `torchtuples`.

## Installation

```bash
pip install pycox
```

## Key Models

| Model | Loss | Notes |
|---|---|---|
| `CoxPH` (DeepSurv) | Cox partial likelihood | Classic deep Cox |
| `CoxCC` | Cox case-control | Memory-efficient approximation |
| `CoxTime` | Cox-Time | Non-proportional relative-risk model |
| `PCHazard` | Piecewise-constant hazard NLL | Continuous-time hazard within intervals |
| `LogisticHazard` | Discrete hazard NLL | Also called Nnet-survival |
| `DeepHitSingle` | DeepHit | Single event, discrete time |
| `DeepHit` | DeepHit | Competing risks |
| `MTLR` | MTLR | Multi-task logistic regression |
| `PMF` | PMF NLL | Direct probability mass over time bins |

## Common Usage

```python
import pycox.models as models
import torchtuples as tt

net = tt.practical.MLPVanilla(in_features, [64, 64], out_features=1)
model = models.CoxPH(net, tt.optim.Adam)
model.fit(x_train, (durations_train, events_train), epochs=100)
model.compute_baseline_hazards()
surv = model.predict_surv_df(x_test)
```

For discrete-time models, use the model's `label_transform()` to fit time-bin cuts
on the training durations, transform every split with those fixed cuts, and set the
network output width to the number of bins. `pycox.evaluation.EvalSurv` provides
time-dependent concordance, IPCW Brier scores, and integrated scores.

## Best For

- Deep learning survival models
- Benchmarking multiple architectures
- Discrete-time and competing risk models
- Research workflows where `torchtuples` owns the training loop

For an entirely custom PyTorch training loop—especially an imaging backbone—compare
with [`torchsurv`](4.3_torchsurv.md). Also check the package's supported dependency
versions before starting a new production environment.
