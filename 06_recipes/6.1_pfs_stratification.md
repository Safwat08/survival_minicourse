# Recipe: PFS Risk Stratification

**Goal:** train a model for progression-free survival (PFS), evaluate its continuous
risk score, and—when a clinical use case requires groups—apply one frozen cutpoint
to an untouched test cohort.

The important separation is:

1. the survival model learns a **continuous score**;
2. development data determine any **group cutpoint**; and
3. the test cohort evaluates both without refitting either one.

## 1. Define the endpoint and prediction origin

Use one composite PFS record per patient; see
[`1.3`](../01_background/1.3_what_is_an_event.md#composite-endpoints):

- `T` — time from the stated origin to the earliest of progression or death, otherwise the censoring time
- `E` — `1` if progression or death occurred and `0` if censored

No event-type label or second event time is needed for composite PFS. If the goal is
to predict progression while treating death as a distinct competing event, redefine
the endpoint and use the methods in
[`1.4`](../01_background/1.4_multi_events_and_competing_risks.md).

Before splitting, confirm that images and clinical predictors were available by the
prediction origin. Split by patient—and by site when testing geographic
generalization—before learning preprocessing, feature selection, or normalization.

## 2. Train the continuous risk model

For a custom PyTorch imaging model, the output is one log-relative-hazard value per
patient. A Cox loss is a defensible baseline:

```python
import torch
from torchsurv.loss import cox

log_hz = model(x_train).reshape(-1)
loss = cox.neg_partial_log_likelihood(log_hz, E_train, T_train)
loss.backward()
```

Do not add a custom log-rank loss merely to make a development KM plot look more
separated. It optimizes the later visualization, introduces another weight to tune,
and can overfit the chosen grouping rule. Start with the survival objective and
evaluate the continuous score honestly.

The exact Cox loss uses full risk sets. If the imaging model requires mini-batches,
use a risk-set-aware strategy or a case-control method rather than assuming an
ordinary random mini-batch reproduces the full loss; see
[`2.1`](../02_loss_functions/2.1_cox_ph_loss.md#key-takeaways-for-applied-ml-scientists).

After model selection is complete, collect scores without augmentation:

```python
model.eval()
with torch.no_grad():
    r_train = model(x_train).cpu().numpy().ravel()
    r_valid = model(x_valid).cpu().numpy().ravel()
    r_test = model(x_test).cpu().numpy().ravel()
```

## 3. Choose one cutpoint on development data

Prefer a clinically pre-specified threshold or a threshold validated in earlier
work. If none exists, the median **training** score is a transparent, outcome-free
baseline:

```python
import numpy as np

tau = np.median(r_train)
```

If the project specifically requires outcome-guided cutpoint selection, use a
separate validation set. The following exploratory max-log-rank search excludes
extreme splits, then freezes the selected value:

```python
from lifelines.statistics import logrank_test
import numpy as np

candidates = np.unique(np.quantile(r_valid, np.linspace(0.10, 0.90, 81)))
best_stat, tau = -np.inf, None

for candidate in candidates:
    high = r_valid >= candidate
    if E_valid[high].sum() == 0 or E_valid[~high].sum() == 0:
        continue
    result = logrank_test(
        T_valid[high], T_valid[~high],
        event_observed_A=E_valid[high],
        event_observed_B=E_valid[~high],
    )
    if result.test_statistic > best_stat:
        best_stat, tau = result.test_statistic, candidate

if tau is None:
    raise ValueError("No candidate produced events in both validation groups.")
```

This validation p-value is not a valid significance result because many cutpoints
were searched. Record the search grid and the resulting `tau`, and do not adjust it
after looking at test outcomes. For small datasets, use nested cross-validation to
repeat the entire model-and-cutpoint procedure.

## 4. Evaluate once on the test cohort

First evaluate the **continuous** score. This preserves information discarded by a
high/low split:

```python
from sksurv.metrics import concordance_index_censored

c_index = concordance_index_censored(
    E_test.astype(bool), T_test.astype(float), r_test
)[0]
print(f"test C-index = {c_index:.3f}")
```

Then apply the frozen cutpoint and draw the descriptive KM curves:

```python
from lifelines import KaplanMeierFitter
from lifelines.plotting import add_at_risk_counts
from lifelines.statistics import logrank_test
import matplotlib.pyplot as plt

high = r_test >= tau
if high.sum() == 0 or (~high).sum() == 0:
    raise ValueError("Frozen cutpoint creates an empty test group.")

result = logrank_test(
    T_test[high], T_test[~high],
    event_observed_A=E_test[high],
    event_observed_B=E_test[~high],
)
print(f"test log-rank p = {result.p_value:.4g}")

ax = plt.subplot()
km_high = KaplanMeierFitter().fit(
    T_test[high], E_test[high], label="High risk"
)
km_low = KaplanMeierFitter().fit(
    T_test[~high], E_test[~high], label="Low risk"
)
km_high.plot_survival_function(ax=ax, ci_show=True)
km_low.plot_survival_function(ax=ax, ci_show=True)
add_at_risk_counts(km_high, km_low, ax=ax)
```

Report the test-set C-index with a confidence interval, group sizes and event counts,
the frozen cutpoint rule, KM confidence bands and at-risk counts, and the log-rank
result. If the model produces survival probabilities, also report the Brier score
and horizon-specific calibration. A small p-value alone is neither an effect size
nor evidence of clinical utility.

## Common failure modes

1. **Selecting or adjusting `tau` on the test set** — this makes the displayed separation optimistic.
2. **Reporting the validation max-log-rank p-value** — the search itself invalidates the naive p-value.
3. **Using only the grouped result** — dichotomization discards information; keep continuous-score metrics primary.
4. **Computing preprocessing before the split** — leaked normalization or feature selection can inflate every downstream metric.
5. **Treating composite PFS as competing risks** — death is already part of the PFS event; use event types only when the scientific question distinguishes causes.
6. **Claiming treatment benefit from prognosis** — a high-risk group identifies poorer prognosis, not which treatment will help.
