Recipe: PFS Risk Stratification

Goal: train a model that predicts progression-free survival, then split patients into high- vs low-risk groups whose KM curves separate cleanly.

This recipe is the connective tissue between sections — it runs the chain 2.1 + log-rank loss → threshold selection → 3.3 end-to-end.

The shape of the data

One composite PFS endpoint per patient — see 1.3:

  • T — time to the earliest of {progression, death}, else censoring time
  • E — 1 if progression or death occurred, 0 if censored

No event-type labels, no second time. This is not competing risks.

Stage 1 — Train a risk model (produces a score, not a split)

The model emits a continuous risk score; higher = worse PFS. For a custom 3D-imaging model you own the forward pass, so use the differentiable Cox loss from torchsurv (see 04_packages/4.3_torchsurv.md). Because stratification is the goal, add the log-rank loss as an auxiliary term (a differentiable relaxation of the log-rank test that rewards separating the risk groups’ survival curves):

from torchsurv.loss import cox

log_hz = model(x)                                  # one log-hazard per patient
loss = cox.neg_partial_log_likelihood(log_hz, events, times)
loss = loss + lam * logrank_aux(log_hz, events, times)   # lam ~ 0.1–0.5; aux is your custom smooth log-rank

(For a tabular baseline, lifelines.CoxPHFitter or sksurv.CoxPHSurvivalAnalysis give the same kind of score — see 2.1.)

After training, get scores for both splits:

r_train = model(x_train).detach().cpu().numpy().ravel()
r_test  = model(x_test ).detach().cpu().numpy().ravel()

Stage 2 — Fit the cutpoint on TRAIN, then freeze it

The split happens here, not in the model. Search the threshold that maximizes the log-rank statistic on the training scores — see threshold selection:

from lifelines.statistics import logrank_test
import numpy as np

best_stat, tau = 0, None
for c in np.percentile(r_train, range(10, 91)):
    hi = r_train >= c
    res = logrank_test(T_train[hi], T_train[~hi], E_train[hi], E_train[~hi])
    if res.test_statistic > best_stat:
        best_stat, tau = res.test_statistic, c

tau is now frozen. A median cutoff (tau = np.median(r_train)) is the safe baseline — see threshold selection.

Stage 3 — Apply the frozen cutpoint to TEST and evaluate

This is the only honest significance test — the cutpoint was never tuned on these patients:

from lifelines import KaplanMeierFitter
from lifelines.plotting import add_at_risk_counts
import matplotlib.pyplot as plt

hi = r_test >= tau                                  # frozen tau from Stage 2

res = logrank_test(T_test[hi], T_test[~hi], E_test[hi], E_test[~hi])
print(f"test log-rank p = {res.p_value:.4f}")       # the money number

ax = plt.subplot()
km_hi = KaplanMeierFitter().fit(T_test[hi],  E_test[hi],  label="High risk")
km_lo = KaplanMeierFitter().fit(T_test[~hi], E_test[~hi], label="Low risk")
km_hi.plot_survival_function(ax=ax); km_lo.plot_survival_function(ax=ax)
add_at_risk_counts(km_hi, km_lo, ax=ax)

Also report the underlying ranking quality — C-index (3.1) and time-dependent AUC (3.4) — so a good split isn’t masking a poorly-discriminating model.

The three ways this goes wrong

  1. Re-deriving tau on the test set — guarantees an optimistic split that fails on external data. Freeze it on train.
  2. Reporting the training max-log-rank p-value as significance — invalid (multiple testing over candidate cutpoints). Only the held-out p-value counts. See threshold selection.
  3. Treating PFS as two events — death is already inside PFS; competing risks is a different question. See 1.3 and 1.4.
