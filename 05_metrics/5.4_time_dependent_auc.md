# Time-Dependent AUC

Standard AUC applies to binary outcomes. Time-dependent AUC extends this to survival data by defining cases and controls at each time point.

## Definition (Incident/Dynamic)

At time t:
- **Cases**: subjects with T_i ∈ [t-Δ, t] and δ_i=1 (event occurred near t)
- **Controls**: subjects with T_j > t (still at risk at t)

```
AUC(t) = P(r_i > r_j | i is case at t, j is control at t)
```

## Cumulative/Dynamic vs. Incident/Dynamic

- **Cumulative/Dynamic**: cases are all events up to t (most common)
- **Incident/Dynamic**: cases are events in a window around t

## Mean AUC

Weighted average of AUC(t) over time, using event density as weights. Provides a single summary statistic comparable to the C-index.

## Advantages over C-index

- Time-resolved discrimination — identifies at which time points the model performs well
- Better suited for comparing models at clinically relevant time horizons

## Implementation

**scikit-survival** — cumulative/dynamic AUC over a vector of time horizons (see [`references/6.4_scikit_survival.md`](../references/6.4_scikit_survival.md)):

```python
from sksurv.metrics import cumulative_dynamic_auc
import numpy as np

times = np.percentile(time_test[event_test], [25, 50, 75])     # within follow-up range
auc, mean_auc = cumulative_dynamic_auc(y_train, y_test, risk_scores, times)
```

**torchsurv** — AUC as a PyTorch metric for in-loop, time-resolved monitoring (see [`references/6.3_torchsurv.md`](../references/6.3_torchsurv.md)):

```python
from torchsurv.metrics.auc import Auc
auc = Auc()(log_hz, events, times, new_time=eval_times)
```

> **Same time-window caveat as Brier.** Both rely on IPCW censoring weights, so evaluation `times` must stay inside the observed follow-up. The time horizon you pick here is also the natural one for the [`04_thresholds/4.2_youden_threshold.md`](../04_thresholds/4.2_youden_threshold.md) cutoff.
