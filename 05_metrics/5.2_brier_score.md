# Brier Score

The Brier score assesses **calibration** — how close predicted survival probabilities are to observed outcomes.

## Definition

At a fixed time t:

```
BS(t) = (1/n) Σ_i [ S(t | x_i) - I(T_i > t) ]²
```

where I(T_i > t) = 1 if the subject survived beyond t, 0 otherwise.

## Inverse Probability of Censoring Weighting (IPCW)

Raw Brier score is biased with censored data. The IPCW-corrected version reweights subjects by the inverse probability of being uncensored:

```
BS_IPCW(t) = (1/n) Σ_i w_i * [ S(t | x_i) - I(T_i > t) ]²
```

## Integrated Brier Score (IBS)

Averages BS(t) over a time range [t_min, t_max]:

```
IBS = ∫ BS(t) dt / (t_max - t_min)
```

Summarizes calibration across all time points.

## Interpretation

- Lower is better; 0 = perfect, 0.25 = uninformative (null model)
- Compare against the Kaplan-Meier baseline (no covariates)

## Implementation

**scikit-survival** is the reference implementation. Both functions take the *training* and *test* structured arrays (for the IPCW censoring estimate) plus the model's predicted survival probabilities `S(t)` at each evaluation time (see [`references/6.4_scikit_survival.md`](../references/6.4_scikit_survival.md)):

```python
from sksurv.metrics import brier_score, integrated_brier_score
import numpy as np

times = np.percentile(time_test[event_test], [25, 50, 75])      # stay inside the followed-up range
surv_prob = np.row_stack([fn(times) for fn in model.predict_survival_function(X_test)])

t, bs = brier_score(y_train, y_test, surv_prob, times)          # BS at each time
ibs   = integrated_brier_score(y_train, y_test, surv_prob, times)  # single summary
```

**torchsurv** provides a differentiable/GPU version for in-loop evaluation (see [`references/6.3_torchsurv.md`](../references/6.3_torchsurv.md)); **auton-survival** reports IBS with bootstrapped confidence intervals (see [`references/6.5_auton_survival.md`](../references/6.5_auton_survival.md)).

> **Pitfall — evaluation times.** Choosing `times` beyond the last observed event makes the IPCW weights explode and the score meaningless. Keep evaluation times strictly inside the follow-up window, and never let `times` include the maximum follow-up time.

> **Brier complements, not replaces, the C-index.** A model can rank perfectly (high C-index) yet be badly calibrated (high Brier), and vice versa. Report both — discrimination *and* calibration. See [`05_metrics/5.1_c_index.md`](5.1_c_index.md).
