# Concordance Index (C-index)

The C-index is the most common discrimination metric in survival analysis. It measures how well the model ranks patients by risk.

## Definition

```
C = P(r_i > r_j | T_i < T_j, δ_i = 1)
```

The probability that a higher-risk subject experiences the event before a lower-risk subject, among all comparable pairs.

## Interpretation

| C-index | Interpretation |
|---|---|
| 0.5 | Random ranking (no discrimination) |
| 0.6–0.7 | Moderate |
| 0.7–0.8 | Good |
| > 0.8 | Strong |

## Comparable Pairs

A pair (i, j) is comparable if:
- T_i < T_j and δ_i = 1 (i had the event first)

Ties and censored-before-event pairs are handled differently across implementations.

## Harrell vs. Uno (IPCW)

There are two C-indices in common use:

- **Harrell's C** — counts all comparable pairs. Simple, but becomes optimistically biased as censoring increases.
- **Uno's C (IPCW)** — reweights pairs by the inverse probability of censoring, giving a consistent estimate under heavier censoring. Prefer it when censoring is non-trivial; it needs the *training* survival distribution to estimate the censoring weights.

## Implementation

**lifelines** — Harrell's C. Note the sign: it expects a quantity that *increases with survival time*, so pass the **negative** risk score (see [`references/6.2_lifelines.md`](../references/6.2_lifelines.md)):

```python
from lifelines.utils import concordance_index
c = concordance_index(event_times, -risk_scores, event_observed)
```

**scikit-survival** — both estimators; the metrics gold standard in the ecosystem (see [`references/6.4_scikit_survival.md`](../references/6.4_scikit_survival.md)):

```python
from sksurv.metrics import concordance_index_censored, concordance_index_ipcw

harrell = concordance_index_censored(event_test, time_test, risk_scores)[0]
uno     = concordance_index_ipcw(y_train, y_test, risk_scores, tau=t_max)[0]
```

**torchsurv** — GPU-compatible, for in-loop monitoring of a PyTorch model (see [`references/6.3_torchsurv.md`](../references/6.3_torchsurv.md)):

```python
from torchsurv.metrics.cindex import ConcordanceIndex
c = ConcordanceIndex()(log_hz, events, times)
```

## Limitations

- Sensitive to censoring patterns
- Does not assess calibration
- A single number — does not capture time-varying discrimination
