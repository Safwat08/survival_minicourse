# Censored Data

Censoring occurs when the exact event time is unknown for some subjects. It is the defining challenge of survival analysis.

## Types of Censoring

- **Right censoring** (most common): Subject exits before the event — lost to follow-up, or study ends
- **Left censoring**: Event occurred before observation began
- **Interval censoring**: Event is known to have occurred within a time window

## Why It Matters

Discarding censored observations introduces bias. Survival models account for censoring by treating censored subjects as contributing information *up to* their last known follow-up time.

## Notation

Each subject has:
- `T` — observed time (event time or censoring time)
- `δ` — event indicator (1 = event occurred, 0 = censored). In code this is usually a column or array named `E` or `event` — the two notations refer to the same thing.

## How Packages Encode (T, δ)

The same two columns, but the expected container differs per package — a frequent source of silent bugs:

| Package | Format |
|---|---|
| lifelines | two columns in a DataFrame: `duration_col="T"`, `event_col="E"` |
| pycox / torchsurv | parallel arrays/tensors: `(durations, events)` |
| scikit-survival | a single **structured** array, event first, as `bool` |

```python
# scikit-survival's required structured-array format
import numpy as np
y = np.array([(bool(e), t) for e, t in zip(events, times)],
             dtype=[("event", bool), ("time", float)])
```

Getting `δ` backwards (treating censored as events) silently corrupts every downstream estimate — verify the indicator convention before fitting. See [`04_packages/4.4_scikit_survival.md`](../04_packages/4.4_scikit_survival.md).
