# Censored Data

Censoring occurs when the exact event time is unknown for some subjects. It is the defining challenge of survival analysis.

## Types of Censoring

- **Right censoring** (most common): Subject exits before the event — lost to follow-up, or study ends
- **Left censoring**: Event occurred before observation began
- **Interval censoring**: Event is known to have occurred within a time window

## Why It Matters

Discarding censored observations introduces bias. Survival models account for censoring by treating censored subjects as contributing information *up to* their last known follow-up time.

## Notation

Let $T^*$ be the true event time and $C$ the censoring time. With right censoring,
the data contain:

- $T = \min(T^*, C)$ — the observed follow-up time
- $\delta = \mathbf{1}(T^* \le C)$ — the event indicator (1 = event observed, 0 = censored)

In code, $\delta$ is usually stored in a column or array named `E` or `event`.

## The assumption behind standard methods

Kaplan–Meier, Cox regression, and the IPCW metrics later in the course usually
assume **independent (non-informative) censoring**. For an unadjusted Kaplan–Meier
curve, censoring should be independent of event time in that group. Regression can
use a conditional version of the assumption: after accounting for included
covariates, censoring should not carry additional information about event time.

Administrative censoring at a planned study end is often plausible. Loss to
follow-up can be informative—for example, if rapidly deteriorating patients stop
attending visits. Standard methods cannot repair that bias automatically; examine
censoring patterns and consider sensitivity analyses or models for informative
censoring when the assumption is doubtful.

> **Censoring is not a competing event.** Censoring hides the future event time. A
> competing event makes the event of interest impossible. The latter requires the
> methods in [`1.4`](1.4_multi_events_and_competing_risks.md).

## How Packages Encode (T, δ)

The same two columns, but the expected container differs per package — a frequent source of silent bugs:

| Package | Format |
|---|---|
| lifelines | two columns in a DataFrame: `duration_col="T"`, `event_col="E"` |
| pycox / torchsurv | parallel arrays/tensors: `(durations, events)`; check required dtypes and shapes |
| scikit-survival | a single **structured** array, event first, as `bool` |

```python
# scikit-survival's required structured-array format
import numpy as np
y = np.array([(bool(e), t) for e, t in zip(events, times)],
             dtype=[("event", bool), ("time", float)])
```

Getting $\delta$ backwards (treating censored observations as events) silently
corrupts every downstream estimate. Before fitting, verify the indicator convention,
time units, absence of impossible times, and alignment between rows of `X`, `T`, and
`E`. See [`04_packages/4.4_scikit_survival.md`](../04_packages/4.4_scikit_survival.md).
