# Multi-Events and Competing Risks

In many clinical settings, patients are at risk for more than one event type, and the occurrence of one may preclude the other.

## Competing Risks

A **competing risk** is an event that prevents the event of interest from occurring. Example: a patient dies before tumor recurrence — death "competes" with recurrence.

Ignoring competing risks leads to overestimation of the cumulative incidence function (CIF).

## Multi-Event Survival

- **Cause-specific hazard models**: one model per event type
- **Sub-distribution hazard (Fine-Gray)**: directly models CIF for each event
- **DeepHIT**: models a joint distribution over event type and time

## Tooling

| Need | Tool |
|---|---|
| Estimate the CIF correctly (descriptive) | `lifelines.AalenJohansenFitter` — see [`references/6.2_lifelines.md`](../references/6.2_lifelines.md) |
| Deep competing-risks model (discrete time) | `pycox.models.DeepHit` — see [`references/6.1_pycox.md`](../references/6.1_pycox.md) and [`03_loss_functions/3.4_deephit_loss.md`](../03_loss_functions/3.4_deephit_loss.md) |
| Deep competing-risks model (parametric mixture) | `auton_survival` DSM — see [`references/6.5_auton_survival.md`](../references/6.5_auton_survival.md) |

The Aalen-Johansen estimator is the competing-risks analogue of Kaplan-Meier — use it instead of `1 - S(t)` for the CIF:

```python
from lifelines import AalenJohansenFitter
ajf = AalenJohansenFitter()
ajf.fit(T, event_observed, event_of_interest=1)   # event labels: 0 = censored, 1, 2, ...
ajf.plot()
```

## Practical Notes

- Always check whether competing risks are present in your dataset
- Multi-event models require event-type labels in addition to (T, δ)
- Plain Kaplan-Meier (`1 - S(t)`) overestimates the CIF under competing risks — use Aalen-Johansen
