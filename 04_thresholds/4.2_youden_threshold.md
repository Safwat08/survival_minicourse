# Youden Index Threshold

The Youden index selects the threshold that maximizes the combined sensitivity and specificity for a binary outcome.

## Definition

```
J = max_τ [ Sensitivity(τ) + Specificity(τ) - 1 ]
```

Equivalent to maximizing the vertical distance from the ROC curve to the diagonal.

## In Survival Analysis

To apply the Youden index in a survival context:

1. Binarize the outcome at a clinically relevant time point (e.g., 2-year recurrence)
2. Compute time-dependent ROC at that time point
3. Find the threshold maximizing sensitivity + specificity

## Properties

- Grounded in diagnostic accuracy
- Clinically interpretable (balances false positives and false negatives)
- Depends on the chosen time point

## Implementation

Binarize at the horizon, then read the Youden-optimal cutoff off the ROC. The honest version uses IPCW-weighted time-dependent ROC (`scikit-survival`, see [`references/6.4_scikit_survival.md`](../references/6.4_scikit_survival.md)) so censored subjects before the horizon are handled correctly:

```python
from sksurv.metrics import cumulative_dynamic_auc
import numpy as np

# 1. time-dependent ROC at the horizon (also gives the AUC, see 05_metrics/5.4)
auc, _ = cumulative_dynamic_auc(y_train, y_test, risk_scores, [horizon])
```

A simpler approximation, valid only when there is little censoring before the horizon, binarizes the outcome and uses `sklearn`'s ROC:

```python
from sklearn.metrics import roc_curve

label = (time <= horizon) & event          # had the event by the horizon
fpr, tpr, thr = roc_curve(label, risk_scores)
tau = thr[np.argmax(tpr - fpr)]            # Youden's J = max(TPR - FPR)
```

> Fit `tau` on training data, then apply that fixed value to the test set — see [`04_thresholds/4.1_median_threshold.md`](4.1_median_threshold.md) for the same train→test discipline.

## Limitations

- Sensitive to event prevalence
- Requires selecting a fixed time horizon
- Can overfit to the training set — validate on held-out data
