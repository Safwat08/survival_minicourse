# Max Log-Rank Training Cutoff

The max log-rank cutoff finds the threshold that maximizes the log-rank statistic between the resulting high- and low-risk groups.

## Method

1. For each candidate threshold τ across the risk score distribution:
   - Split patients into high-risk (r ≥ τ) and low-risk (r < τ)
   - Compute the log-rank test statistic between the two KM curves
2. Select τ* = argmax_τ log-rank statistic

## Properties

- Directly optimizes survival curve separation
- Does not require a fixed time horizon (unlike Youden)
- Produces the most statistically significant KM split

## Critical Warning: Multiple Testing

Testing many thresholds inflates the Type I error rate. The resulting p-value is **not valid** — the threshold must be validated on a held-out test set or independent cohort.

**Never report the training log-rank p-value as a significance test.**

## Implementation

```python
from lifelines.statistics import logrank_test
import numpy as np

best_stat, best_tau = 0, None
for tau in np.percentile(risk_scores, range(10, 91)):
    mask = risk_scores >= tau
    result = logrank_test(T[mask], T[~mask], E[mask], E[~mask])
    if result.test_statistic > best_stat:
        best_stat, best_tau = result.test_statistic, tau
```

Uses `lifelines.statistics.logrank_test` (see [`references/6.2_lifelines.md`](../references/6.2_lifelines.md)). The differentiable training-loss counterpart of this cutoff search is the log-rank loss (summarized in [`3.5`](../03_loss_functions/3.5_loss_comparison.md)); fit `best_tau` on train, then validate the split on held-out data per [`4.1_median_threshold.md`](4.1_median_threshold.md).
